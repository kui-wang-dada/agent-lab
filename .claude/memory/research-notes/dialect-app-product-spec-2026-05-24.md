## 0. 我的判断（先说结论）

**当前需求不可直接动工**。原话里有三个高风险模糊点（"所有方言"/"识别"/"文案"），任何一个理解错都会做错产品。建议先用本文档的「需求澄清清单」回到甲方那对齐，**拿到至少 6 项答复前不要写代码、不要承诺工期**。

**强烈警示**：「所有方言」是不可兑现承诺。中国境内汉语方言学界普遍承认 7-10 大方言区，下分 100+ 次方言、500+ 土语；产业级 ASR 目前**真正能跑的方言**约 10-15 种（详见调研报告）。直接接需求会埋大坑。

## 1. 需求澄清清单（必须问甲方的 10 个问题）

按重要性排序，每个含「为什么决定产品形态」+「常见答案 + 推荐」。

### Q1【最关键】「所有方言」具体覆盖哪些？

- **为什么决定产品形态**：覆盖范围直接定 ASR 选型（开源模型 / 云 API / 自训），也定项目难度与报价。10 种和 50 种差一个数量级的工作量。
- **选项**：
  - A. 中国汉语方言主流前 5（粤/闽南/吴/川/客家）—— **推荐起步**
  - B. 中国汉语方言 + 部分少数民族语言（藏/维/蒙等）—— 难度 ×3，需调云端 API
  - C. 全球任意语言方言 —— 现实里只能"识别成普通话"+ 翻译兜底
- **推荐**：MVP 先 A，写进合同；B/C 作为 v2 路线图。

### Q2【最关键】"生成文案"是什么文案？

- **为什么决定产品形态**：决定后端是 LLM prompt 模板还是 RAG 系统，决定 UI 是单 textarea 还是多模板选择器，也决定是 toC 工具还是 toB SaaS。
- **选项**：
  - A. **短视频脚本/口播稿**（抖音/小红书创作者用）—— 离消费场景最近
  - B. **方言 → 普通话转写 + 文字稿**（媒体/记者/字幕用）—— 工具属性强
  - C. **客服话术/营销文案**（B 端用）—— 企业付费意愿高
  - D. **朋友圈/IM 短句**（C 端社交用）—— 用户量大但难收钱
- **推荐**：让甲方挑 1 个主场景，其他作为 v2 模板。最值得做的是 A 或 C。**避免"我全都要"**。

### Q3 目标用户是谁？

- **为什么决定产品形态**：toC 工具（小程序/App）vs toB SaaS（Web 后台/API）的产品形态、获客成本、单价天差地别。
- **选项**：
  - A. **C 端创作者**（地方网红 / 方言段子手 / 银发主播）
  - B. **B 端企业**（地方电商客服、方言电台、方言培训机构）
  - C. **政府/文旅**（方言保护、地方文化数字化）
  - D. **银发/长辈**（家庭沟通辅助）
- **推荐**：让甲方明确 1 个主用户。**Kevin 视角**：B 优先（付费意愿强、需求清晰、定制空间大）；C 谨慎（资质门槛高、回款慢，业务边界与 Kevin 的"不接政务"硬约束冲突）。

### Q4 单次输入时长 + 实时性要求？

- **为什么决定产品形态**：决定 ASR 是流式（streaming）还是离线（batch）、决定后端架构、决定带宽/费用模型。
- **选项**：
  - A. 短句 10-30s（开口即得）—— 实时流式
  - B. 中段 1-3min（说一段话）—— 录完上传
  - C. 长录音 5-30min（采访/会议）—— 异步任务 + 进度
- **推荐**：MVP 锁定 B（最常用，技术中等）。A 和 C 在 v2 加。

### Q5 是否需要保留方言原文转写？

- **为什么决定产品形态**：保留 = 需要做"方言文字"输出（粤语字、闽南语白话字等专业工程，难度极高）；不保留 = 直接转普通话 + 文案，难度大幅下降。
- **选项**：
  - A. 只要"听懂含义"，输出普通话文案 —— **推荐**
  - B. 要方言原文（粤语字等）+ 普通话对照
  - C. 不要转写，只要"基于方言意图"生成文案
- **推荐**：A。直接说"方言文字化属于学术工程，不在本期范围"。

### Q6 离线还是在线？

- **为什么决定产品形态**：离线 = 端侧 SDK（移动端集成，需模型压缩，工程量极大）；在线 = 云 API（标准 Web 后台，2 周可上线）。
- **选项**：
  - A. **在线 SaaS**（小程序 / Web）—— 推荐
  - B. 端侧 SDK（iOS/Android 本地推理）—— 工程量 ×5
  - C. 私有化部署（B 端客户内网）—— 加 ¥10w+ 部署费
- **推荐**：A 起步。私有化作为 B 端可选包。

### Q7 商业模式？

- **为什么决定产品形态**：决定产品有没有支付/订阅/计费/对账模块，影响 30%+ 的工程量。
- **选项**：
  - A. **甲方自己运营，Kevin 只交付一次性软件**（外包模式）—— 推荐
  - B. 订阅制（¥XX/月，按次/按时长扣额度）
  - C. B 端授权 + API 计费
  - D. 免费引流 + 增值服务
- **推荐**：先问甲方"这是给你自己用，还是要卖给终端用户？"。如果是 A，Kevin 报固定价；B/C 涉及支付通道、对账、退款、客服，加 30-50% 工时和费用。

### Q8 平台 / 落地形态？

- **为什么决定产品形态**：小程序 vs App vs Web 后台技术栈完全不同；麦克风权限、录音格式、上传方式都不同。
- **选项**：
  - A. **微信小程序**（C 端门槛低、抖音渠道也可投放）—— 推荐
  - B. Web 后台（B 端）
  - C. iOS/Android App（端侧 + 商店审核）
  - D. 公众号 / 抖音小程序
- **推荐**：A。小程序生态成熟、录音 API 标配、Kevin 有微信生态实战。

### Q9 预算范围 + 期望工期？

- **为什么决定产品形态**：决定 MVP 切片选哪个版本。
- **选项**（参考国内 B 端定价）：
  - A. ¥3-5w / 3 周（最小可用）
  - B. ¥8-15w / 6-8 周（含多模板 + B 端 SaaS 后台）
  - C. ¥20w+ / 3 月 +（含私有化 / API / 多端）
- **推荐**：先反问甲方"心理预算大概在什么档位"，再给方案。Kevin 别先报价。

### Q10 后期维护 + 模型迭代谁负责？

- **为什么决定产品形态**：ASR 模型每 3-6 月有新版本（SenseVoice / FireRedASR / Qwen3-ASR 都在快速演进），云 API 价格也在变。如果甲方期望"做完不再付钱永远好用"——这是不可能的承诺。
- **选项**：
  - A. 一次性交付，无维护（Kevin 推荐）
  - B. 包 3-6 月 bug 修复（+10% 报价）
  - C. 年度运维包 + 模型升级（+20%/年）
- **推荐**：A。强调"AI 模型 6 月一代，明年要再升级请单独报价"。

## 2. 三种 MVP 切片方案（让甲方选）

> 所有方案前提：在线 SaaS、微信小程序前端、方言转普通话文案、Kevin 单人 + AI 并行写代码。

### 方案 A — 最小可用（¥3-5w / 2-3 周）

- **覆盖**：粤语、闽南语、四川话、上海话、东北话（5 种）
- **文案模板**：1 种主场景（如"短视频口播脚本"或"客服回复"，二选一）
- **核心功能**：
  - 小程序录音上传（单次 ≤2min）
  - 调阿里云 Bailian Fun-ASR（¥0.288/h，方言能力较好）
  - LLM 模板生成（GPT-4o-mini 或 DeepSeek，¥0.001/次级别）
  - 输出文本可复制/下载
- **不含**：用户系统、付费模块、文案模板库、B 端后台、私有化
- **风险**：方言识别准确率不会很高（70-85%），需在 UI 上加"请尽量靠近麦克风/请讲清楚"提示
- **报价拆解**：5 人天产品+设计 + 8 人天开发 + 2 人天联调 = 15 人天 × ¥3000 = ¥4.5w

### 方案 B — 中等（¥8-15w / 6-8 周）【**推荐起步**】

- **覆盖**：方案 A 的 5 种 + 闽东语、客家话、湖南话、河南话、陕西话、温州话、潮汕话、贵州话、山东话、武汉话（共 15 种）
- **文案模板**：3-5 种（短视频脚本 / 客服话术 / 朋友圈短句 / 营销标语 / 字幕）
- **新增**：
  - 用户登录（微信 OAuth）
  - 历史记录（最近 30 条录音 + 生成结果）
  - 模板自定义提示词（用户可改 prompt 调风格）
  - 简单后台（导出 CSV / 总使用量统计）
- **不含**：付费订阅（如需付费，加 ¥2w 接微信支付）、实时流式、长录音、API 开放、私有化
- **风险**：
  - 15 种方言里 5 种以上要靠"模型 + 提示词补救"（小众方言云 API 也覆盖不全），需在 UI 上**明示哪几种是"实验性"**
  - LLM 月成本随用户量上升，月活 100 人内 < ¥200/月，月活 1000+ 需重新核算
- **报价拆解**：8 人天产品+设计 + 20 人天开发 + 4 人天联调 + 3 人天 buffer = 35 人天 × ¥3000 = ¥10.5w（中位）

### 方案 C — 完整（¥20-35w / 3-4 月）

- **覆盖**：30+ 方言（其中 10 种 SLA 保证 + 20 种"尽力而为"）
- **场景**：方案 B 全部 + B 端 SaaS 后台 + 开放 API + 私有化部署包
- **新增**：
  - 实时流式识别（直播/电台场景）
  - 长录音异步任务（≤30min）
  - 多用户/多角色后台（管理员 / 普通用户 / API 用户）
  - 计费系统（按次/按时长，对接微信/支付宝/对公）
  - 模型层混跑（云 API 主力 + 自部署 SenseVoice/Qwen3-ASR 兜底，降本）
  - 私有化部署文档 + 一键脚本
- **不含**：方言文字化（粤语字等）、端侧 SDK、模型自训
- **风险**：
  - 工期长，期间客户需求大概率变化 3+ 次，必须按 dev/learnings.md「CR + OQ + _inflight 三件套」管理
  - 自部署模型上线后运维持续成本 ¥2000-5000/月（GPU 服务器）
- **报价拆解**：60-80 人天，按 ¥3000-5000/天估，¥18-40w 区间

## 3. 推荐方案 B 的用户故事（5 条核心）

> 视角：B 端"地方电商客服主管" + C 端"方言短视频创作者"混合。

1. **作为短视频创作者**，我希望对小程序说一段四川话脚本想法，**以便**它自动转成普通话短视频口播稿，省去我手写脚本时间。
2. **作为电商客服主管**，我希望我的客服对着小程序说粤语回复，**以便**系统生成标准书面客服话术，让新客服减少"听不懂粤语"的培训成本。
3. **作为银发用户的子女**，我希望长辈对着小程序说闽南语，**以便**我能看到普通话翻译 + 一句礼貌回复模板，方便我用普通话回话。
4. **作为方言创作者**，我希望选择"朋友圈短句"模板，**以便**一段口语方言自动变成 3 条精简文案候选供我挑选发布。
5. **作为重度用户**，我希望能看到最近 30 条录音 + 文案记录并复用提示词，**以便**保持创作风格一致。

可选扩展故事：

6. **作为后台管理员**，希望看到本月所有用户的使用次数和热门方言/模板，**以便**判断哪些方言/模板可砍可加。
7. **作为试用用户**，我希望前 5 次免费，**以便**先看效果再决定要不要订阅（如商业模式选订阅，对应 Q7 答 B 时启用）。

## 4. 关键风险 + 决策建议

### 风险 1：「所有方言」是不可兑现承诺

**对甲方的话术**（建议 Kevin 这么说）：

> "中国方言学界承认 7-10 大方言区，下分 100+ 次方言。**真正能做到识别准确率 ≥80% 的方言全行业不超过 15 种**（含我们要用的阿里云 / SenseVoice / 通义听悟最强组合）。其余方言要么没有训练数据、要么市场太小 ASR 厂商不投入。建议我们按"主流 5 种 → 扩到 15 种 → 30+ 种但其中 20 种标实验性"分三档走，先签第一档（方案 A 或 B），上线后看用户实际用哪些方言再迭代加。"

**为什么这样说**：保护合同条款不被"所有方言"字面绑死、保护未来甲方追责空间、给 Kevin 留迭代空间。

### 风险 2：文案模板边界不清会无限改稿

**对策**：合同里写死"v1.0 上线时 N 个模板，每个模板 1 套提示词。后续模板新增、风格调优属于变更，按单独工单计费"。参考 dev/learnings.md 的 CR 单做法。

### 风险 3：方言识别准确率不稳定 → 用户投诉

**对策**：
- UI 明示"AI 识别可能有误，请核对再用"
- 提供"重录"和"手动修改文字稿"按钮
- 后台监控低置信度结果，每周复盘 case，迭代提示词
- **合同条款**：交付以"特定测试集准确率 ≥ X%"为验收标准，而非"用户主观满意"

### 风险 4：商业边界与 Kevin 业务规划潜在冲突

参考 `business-plan.md` 和 `kevin-domestic/facts.md`：
- ✅ B 端 SaaS、企业客户、地方电商 —— 符合 Kevin 接单偏好
- ⚠️ 文旅/政务/方言保护项目 —— 资质门槛 + 回款慢，**不接**
- ⚠️ "做完 + 长期运营给 C 端用户" —— 与 Kevin "60% Upwork 主线" 时间分配冲突，建议甲方自己运营

### 推荐报价区间

参考 `kevin-domestic/facts.md` 的"国内定价"段位（待校准，目前用业内中位估）：

- **方案 A**：¥3-5w 一口价
- **方案 B**：¥8-15w 一口价，**Kevin 卖这档最舒服**（与时间分配 + 利润率匹配）
- **方案 C**：¥20-35w 分阶段付款（30/40/30）

**重要原则**（来自 domestic/facts.md）：
- 必签合同（哪怕是朋友介绍）
- 预付 ≥30%
- 工期 buffer 1.3-1.5 倍
- 超范围需求 = 单独报价（防无限改稿）
- 国内一口价 + 明确 SOW 比按小时计价更省心

```ts
// 录音 → 识别 → 生成 一次性请求
type DialectGenerateRequest = {
  audio: {
    format: 'mp3' | 'wav' | 'm4a'
    durationSec: number  // ≤ 120 for MVP
    base64OrUrl: string
  }
  dialectHint?: DialectCode  // 用户可选指定，否则自动检测
  templateId: TemplateId     // 如 'short_video_script' | 'customer_service' | ...
  customPromptOverride?: string  // v1 中等版才开放
}

type DialectGenerateResponse = {
  recognized: {
    rawTextMandarin: string       // 转成普通话的原文
    confidence: number            // 0-1
    detectedDialect: DialectCode
  }
  generated: {
    text: string                  // 最终文案
    alternatives?: string[]       // 候选 2-3 条
    templateUsed: TemplateId
  }
  metadata: {
    asrProvider: 'aliyun-bailian' | 'sensevoice-self-host'
    llmProvider: 'gpt-4o-mini' | 'deepseek-v3'
    costCNY: number               // 单次成本（后台监控用，不展示给用户）
    latencyMs: number
  }
}

type DialectCode =
  | 'cantonese' | 'min-nan' | 'wu' | 'sichuan' | 'dongbei'  // MVP 5 种
  | 'hakka' | 'hunan' | 'henan' | 'shaanxi' | 'wenzhou'
  | 'chaoshan' | 'guizhou' | 'shandong' | 'wuhan' | 'min-dong'  // 中等版加 10
  | 'auto'

type TemplateId =
  | 'short_video_script'
  | 'customer_service_reply'
  | 'wechat_moments'
  | 'marketing_copy'
  | 'subtitle'
```
